# 📇 Java Web Contact Manager (CRUD)

## 📌 About this Project
This repository contains a fully functional Web Application for managing contacts, developed as a hands-on project to solidify core Java web concepts. 

As a professional accustomed to complex ERP business rules, this project represents the technical transition to hands-on web backend development. It demonstrates a clear understanding of the **JavaBean pattern**, web form integrations, data storage, and the practical application of **HTTP Verbs** to build a complete CRUD (Create, Read, Update, Delete) cycle.

## 🚀 Features & Implementation

The application was built progressively, covering the following architectural and developmental milestones:

*   **Domain Modeling:** Creation of the `Contact` entity strictly following the **JavaBean pattern** (encapsulation, default constructors, getters/setters) to ensure compatibility with enterprise frameworks.
*   **Data Storage Configuration:** Implementing the persistence layer logic to securely store and retrieve contact records.
*   **Read (Listing):** Dynamically rendering stored contacts directly onto the web page interface.
*   **HTTP Verbs & Routing:** Deep dive into web protocols, mapping user actions to appropriate HTTP methods (`GET` for reading, `POST` for registration/updates).
*   **Create (Registration):** Enabling the registration page, capturing web form data, and converting it into Java objects.
*   **Update (Editing):** Building the logic to fetch a specific contact, populate the editing form, and process the update request.
*   **Delete (Removal):** Safely removing contact entities from the storage layer.

## 🛠️ Tech Stack & Concepts
*   **Language:** Java
*   **Architecture:** Client-Server Web Architecture
*   **Core Concepts:** OOP, JavaBean Pattern, CRUD Operations, HTTP Protocol (Verbs), Web Forms Handling.
